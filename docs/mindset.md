# Mindset

この講座で一番大事な前提です。

## 1. 全部 AI に聞く。全部 AI にやらせる。

最初から自分で正しい手順を知っている必要はありません。

わからないことが出たら、まず AI に聞いてください。

```text
私は非エンジニアです。
この画面で次に何をすればいいか、初心者にもわかるように順番に教えてください。
```

```text
このエラーが出ました。
原因を説明して、次に押すボタン、入力する内容、確認する場所を具体的に教えてください。
```

```text
この作業を私の代わりに進めてください。
途中で私の承認が必要なところだけ、理由を短く説明してください。
```

大事なのは、「自分が全部理解してから進む」ではなく、「AI に聞きながら前に進む」ことです。

## 2. 最新情報は必ずリサーチする

AI 開発ツールは変化が速いです。

- UI が変わる
- 料金が変わる
- 使える model が変わる
- GitHub / Vercel / Codex / Claude の連携方法が変わる
- 便利な MCP や agent ツールが毎月出てくる

だから、古い記事や古い動画をそのまま信じないでください。

特に、AI 驚き屋の情報は玉石混交です。

- すごそうに見えるけど再現できない
- デモでは動くけど実務では不安定
- 課金や権限まわりのリスクが説明されていない
- 実は古い model / 古い UI の話だった

おすすめはこの順番です。

1. 公式 docs を見る
2. Codex / Claude に最新情報を調べさせる
3. 自分で小さく触って試す
4. うまくいった手順を仲間と共有する
5. うまくいかなかったことも共有する

## 3. エンジニアにしかできないと思わない

MVP の最初の一歩は、もうエンジニアだけのものではありません。

もちろん、難しい設計、セキュリティ、課金、データベース、本番運用には専門知識が必要です。

でも、次のことは非エンジニアでも始められます。

- 企画を言語化する
- 画面のたたき台を作る
- ユーザーフローを試す
- LP や簡単な Web アプリを作る
- Vercel に公開する
- ユーザーに見せて反応を見る

最初の MVP の目的は、完璧なプロダクトを作ることではありません。

「この課題は本当にあるのか」「この形ならユーザーが触ってくれるのか」を早く確かめることです。

## 4. AI に任せるが、責任は持つ

全部 AI に聞いて、全部 AI にやらせて構いません。

ただし、次の判断は人間が持ちます。

- 誰の課題を解くのか
- 何を作らないのか
- 課金を有効にするか
- 本番データを扱うか
- 外部 API key を入れるか
- 公開してよい内容か

AI は作業を速くします。
起業家は、何を作るべきか、何を試すべきか、何を公開してよいかを判断します。

## 5. 迷ったらこのプロンプト

```text
私は非エンジニアの起業家です。
MVP を作りたいですが、次に何をすればいいかわかりません。

今の状況:
- やりたいこと:
- できていること:
- 詰まっていること:

次にやるべきことを、初心者でも実行できる手順に分解してください。
できる作業はあなたが進めてください。
私の承認が必要な操作だけ、理由を説明して止まってください。
```
